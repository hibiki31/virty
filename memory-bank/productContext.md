# Product Context - 機能要件・ユーザーワークフロー・UX目標

## プロダクト概要

**Virty** は、libvirtを使用したKVM仮想化基盤のWebベース管理ツールである。
低コストかつ即座にデプロイ可能な仮想化環境の構築・管理を目的とする。
SSH経由でlibvirt APIやAnsibleを利用してノードを管理する。

## コア機能

### 1. ノード管理（Nodes）
- libvirtホストの登録・管理
- SSH経由でのノード接続・状態監視
- ノードスキャン（ドメイン・ネットワーク・ストレージの自動検出）

### 2. 仮想マシン管理（Domains）
- VM一覧表示・詳細表示
- VM作成・起動・停止・削除
- コンソール接続
- フレーバー（CPUコア数・メモリ・ディスク仕様）テンプレートによるVM作成

### 3. ネットワーク管理（Networks）
- 仮想ネットワークの作成・管理
- Open vSwitch (OVS) 対応
- VLAN設定
- isolatedモード・NATモード対応
- ポートグループ管理

### 4. ストレージ管理（Storages）
- ストレージプールの作成・管理
- ディスクボリュームの管理
- リスキャン機能

### 5. イメージ管理（Images）
- ディスクイメージの管理
- ISOイメージのダウンロード（wget）
- cloud-init対応

### 6. プロジェクト管理（Projects）
- リソースのプロジェクト単位でのグルーピング
- マルチテナント的なリソース分離

### 7. フレーバー管理（Flavors）
- VM仕様テンプレート（CPU・メモリ・ディスク）の定義
- VM作成時のテンプレート選択

### 8. ユーザー管理（Users）
- ユーザー登録・認証（JWT）
- 公開鍵管理
- ロールベースアクセス制御

### 9. タスク管理（Tasks）
- 非同期タスクの状態監視
- タスク依存チェーンの可視化
- タスク履歴・結果の確認

### 10. モニタリング（Exporter）
- Prometheusメトリクスエンドポイント
- メトリクス可視化

## ユーザーワークフロー

### VM作成の基本フロー
1. ノードを登録（SSH接続情報）
2. ノードスキャンでリソース検出
3. ストレージプール設定
4. ネットワーク設定
5. フレーバー定義（CPU/メモリ/ディスク仕様）
6. イメージ準備（ISOダウンロードまたはディスクイメージ）
7. VM作成（フレーバー + ネットワーク + ストレージ選択）
8. VM起動・コンソール接続

### 日常運用フロー
1. ダッシュボードで全体状況確認
2. VM一覧で稼働状態確認
3. 起動/停止/再起動操作
4. タスク画面で非同期操作の進捗確認

## UX目標

- **シンプルなUI**: Vuetify3によるマテリアルデザインで直感的な操作
- **リアルタイム性**: タスクの進捗をリアルタイムで把握可能
- **型安全性**: OpenAPI型生成によるフロント・バック間の型整合保証
- **低コスト運用**: Docker Composeによる簡単デプロイ
- **拡張性**: ドメイン分割アーキテクチャによるモジュール追加の容易さ
